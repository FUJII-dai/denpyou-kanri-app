Register Cash テーブル設定手順

問題

アプリケーションが register_cash テーブルにアクセスしようとしていますが、このテーブルがデータベースに存在しません。

エラーメッセージ: relation "public.register_cash" does not exist

解決方法

Supabaseダッシュボードで直接SQLを実行して、register_cashテーブルを作成する必要があります。

1 Supabaseダッシュボードにログイン

Supabaseダッシュボードを開く

プロジェクト「yfsrptorzuwirqgslchk」を選択してください。

2 SQL Editorを開く

左側のメニューから「SQL Editor」を選択してください。

3 新しいSQLクエリを作成

「+ New Query」ボタンをクリックして、新しいSQLクエリを作成してください。

4 SQLスクリプトをコピー＆ペースト

以下のSQLスクリプトをコピーして、エディタに貼り付けてください：

-- レジ金確認テーブル作成スクリプト
-- Supabaseダッシュボードの「SQL Editor」で実行してください

-- テーブル作成
CREATE TABLE IF NOT EXISTS register_cash (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  business_date text NOT NULL UNIQUE,
  starting_amount integer DEFAULT 0,
  coins_amount integer DEFAULT 0,
  next_day_amount integer DEFAULT 0,
  next_day_coins integer DEFAULT 0,
  withdrawals jsonb DEFAULT '[]'::jsonb,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- 行レベルセキュリティの有効化
ALTER TABLE register_cash ENABLE ROW LEVEL SECURITY;

-- ポリシーの作成
CREATE POLICY "Users can read all register_cash"
  ON register_cash
  FOR SELECT
  TO authenticated
  USING (true);

CREATE POLICY "Users can insert register_cash"
  ON register_cash
  FOR INSERT
  TO authenticated
  WITH CHECK (true);

CREATE POLICY "Users can update register_cash"
  ON register_cash
  FOR UPDATE
  TO authenticated
  USING (true)
  WITH CHECK (true);

-- 更新日時トリガーの作成
CREATE OR REPLACE FUNCTION update_register_cash_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_register_cash_updated_at
  BEFORE UPDATE ON register_cash
  FOR EACH ROW
  EXECUTE FUNCTION update_register_cash_updated_at();

-- 初期データの作成（今日と明日の日付用）
INSERT INTO register_cash (business_date, starting_amount, coins_amount, next_day_amount, next_day_coins)
VALUES 
  (to_char(current_date, 'YYYY-MM-DD'), 0, 0, 0, 0),
  (to_char(current_date + interval '1 day', 'YYYY-MM-DD'), 0, 0, 0, 0)
ON CONFLICT (business_date) DO NOTHING;

5 SQLを実行

「Run」ボタンをクリックして、SQLスクリプトを実行してください。

6 アプリケーションを再読み込み

SQLスクリプトが正常に実行されたら、アプリケーションを再読み込みしてください。

アプリケーションを再読み込み

確認方法

テーブルが正常に作成されたことを確認するには：

  1. アプリケーションを再読み込みする
  2. 「レジ金確認」ページにアクセスする
  3. ページが正常に読み込まれ、入力フィールドが表示されることを確認する

テーブルが正常に作成されると、レジ金確認ページが正しく表示され、すべての入力フィールドが機能するようになります。
